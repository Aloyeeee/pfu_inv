Система управління інвентарем Головного управління Пенсійного фонду України в Сумській області

Inventory Management System of the Main Department of the Pension Fund of Ukraine in Sumy Region

Розробка веб-орієнтованої інформаційної системи для автоматизації обліку техніки, витратних матеріалів та контролю переміщень у державній установі.

🔑 Ключові слова

інвентарний облік управління майном веб-система PHP MySQL JavaScript ПФУ бакалаврська робота

📌 Актуальність теми

У сучасних умовах цифровізації державних установ постає необхідність у створенні ефективних інструментів для управління майном. Розроблена система дозволяє автоматизувати облік понад 23 000 одиниць техніки, забезпечує контроль за матеріально-відповідальними особами та генерує всі необхідні звітні документи відповідно до стандартів ДСТУ.

Впровадження такої системи дозволяє:

  • зменшити час на пошук інформації про техніку до 70%;
  • автоматизувати формування звітів для керівництва;
  • забезпечити прозорість переміщень та списань майна;
  • підвищити відповідальність матеріально-відповідальних осіб.

🎯 Мета дослідження

Розробити веб-орієнтовану систему управління інвентарем для Головного управління Пенсійного фонду України в Сумській області, яка забезпечує:

  • прозорий облік усієї техніки та майна;
  • автоматичну генерацію звітних документів (PDF, Excel);
  • контроль за переміщеннями та списанням;
  • розмежування прав доступу користувачів;
  • інтеграцію з існуючою інфраструктурою установи.

📋 Основні завдання

  1. Провести аналіз існуючих систем обліку майна та визначити вимоги до нової системи.
  2. Розробити структуру бази даних для зберігання інформації про техніку, кабінети, переміщення та історію змін.
  3. Створити інтуїтивно зрозумілий інтерфейс для роботи з даними (склад, кабінети, картриджі, ноутбуки).
  4. Реалізувати функції генерації звітних документів у форматах PDF та Excel.
  5. Забезпечити розмежування прав доступу (адміністратор/користувач) та аудит дій.
  6. Оптимізувати продуктивність системи для роботи з великою кількістю даних (23 000+ записів).
  7. Забезпечити адаптивність та кросбраузерну сумісність веб-інтерфейсу.

⚙️ Методологія дослідження

Для реалізації системи використано сучасні веб-технології:

  • PHP 7.4+ – серверна логіка та REST API;
  • MySQL / MariaDB – реляційна база даних з оптимізованими запитами;
  • JavaScript (ES6+) – клієнтська логіка та інтерактивність;
  • HTML5 / CSS3 – семантична розмітка та адаптивний дизайн;
  • Chart.js – візуалізація даних на дашборді;
  • pdfMake – генерація PDF-документів (акти, формуляри);
  • SheetJS (XLSX) – експорт даних у Excel.

Архітектура системи побудована за принципом модульності: кожен функціональний блок (склад, кабінети, картриджі тощо) має окремий модуль як на бекенді, так і на фронтенді. Взаємодія між клієнтом та сервером реалізована через REST API.
